Yeah that's the crux of it. C basically assumes everything *could* be mutable, there's no discipline around it at all. So when you transpile you have to do the analysis work that C never bothered to do:

- **What's actually mutated across loop iterations** vs what's just read
- **What's truly immutable** and can be treated as a constant tensor
- **What's loop-carried state** that needs to be explicitly threaded through the tail recursion

And C code in practice is written with zero consideration for any of this, so the transpiler has to infer it all from usage patterns.
